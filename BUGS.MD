# BUGS.md
## Обнаруженные баги

### BUG-001: Обязательное поле statistics при создании объявления
**Краткое описание:** API требует обязательного наличия всех полей в statistics (likes, viewCount, contacts)
**Шаги воспроизведения:**
1. Отправить POST запрос на /api/1/item без поля statistics или с неполным объектом
**Фактический результат:** 400 с сообщением "поле likes обязательно"
**Ожидаемый результат:** Успешное создание или более информативное сообщение об ошибке
**Серьёзность:** Medium
**Окружение:** https://qa-internship.avito.com

### BUG-002: Нестабильный формат ответа при создании объявления
**Краткое описание:** API возвращает разные форматы ответа при создании объявления
**Шаги воспроизведения:**
1. Отправить POST запрос на /api/1/item с валидными данными
**Фактический результат:** Формат ответа варьируется (объект/массив/строка)
**Ожидаемый результат:** Всегда объект с полями id, sellerId, name, price, statistics, createdAt
**Серьёзность:** Medium
**Окружение:** https://qa-internship.avito.com

### BUG-003: Несоответствие формата создания объявления документации
**Краткое описание:** API возвращает 400 с требованием поля `likes` на верхнем уровне, хотя по документации оно должно быть внутри объекта `statistics`
**Шаги воспроизведения:**
1. Отправить POST /api/1/item с телом согласно коллекции Postman
**Фактический результат:** 400 с сообщением "поле likes обязательно"
**Ожидаемый результат:** 200, объявление создано
**Серьёзность:** Critical (блокирует основную функциональность)
**Окружение:** https://qa-internship.avito.com
