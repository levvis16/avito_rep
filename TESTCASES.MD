# TESTCASES.md

## Создание объявления (POST /api/1/item)

**Позитивные:**
- Создание объявления с валидными данными (sellerID, name, price, statistics) — ожидается 200 и возврат ID
- Создание объявления с заполненной статистикой (likes, viewCount, contacts) — ожидается 200 и сохранение значений

**Негативные:**
- Отрицательная цена — 400
- Нулевая цена — 400
- Пустое название — 400
- Отсутствие sellerID в теле запроса — 400
- Невалидный тип sellerID (строка вместо числа) — 400
- Отсутствие поля statistics — 400
- Неполный объект statistics — 400

---

## Получение объявления по ID (GET /api/1/item/{id})

**Позитивные:**
- Запрос существующего ID — 200, возвращаются данные объявления

**Негативные:**
- Запрос несуществующего ID — 404
- Пустой ID в URL — 400 или 404
- Запрос ID удалённого объявления — 404

---

## Получение объявлений по sellerID (GET /api/1/{sellerID}/item)

**Позитивные:**
- Запрос sellerID с существующими объявлениями — 200, возвращается список
- Запрос sellerID без объявлений — 200, возвращается пустой список
- Создание нескольких объявлений и проверка что все возвращаются в списке

**Негативные:**
- Невалидный sellerID (строка) — 400

---

## Получение статистики по объявлению (GET /api/1/statistic/{id})

**Позитивные:**
- Запрос статистики существующего объявления — 200, возвращаются likes, viewCount, contacts
- Создание объявления с заданной статистикой и проверка что значения совпадают
- Сравнение статистики из API v1 и v2 — данные должны быть идентичны

**Негативные:**
- Запрос статистики несуществующего ID — 404

---

## E2E сценарии

- Полный цикл: создать объявление → получить по ID → получить по sellerID → удалить → проверить что объявление не найдено
- Создание нескольких объявлений одним продавцом и проверка списка
- Идемпотентность: два одинаковых запроса на создание возвращают разные ID

---

## Корнер-кейсы

- Идемпотентность создания — повторный запрос создаёт новое объявление, а не обновляет существующее
- Согласованность версий API — v1 и v2 возвращают одинаковую статистику для одного объявления
- Пустой ID в GET-запросах — корректная обработка ошибки
- Удаление несуществующего объявления — 404

---

## Обнаруженные проблемы

**BUG-003:** API не принимает структуру запроса согласно документации. Сервер ожидает поле likes на верхнем уровне, а не внутри объекта statistics. Из-за этого позитивные тесты на создание объявления падают с ошибкой 400.

Тесты, зависящие от этого бага, помечены как xfail или skipped в коде автотестов.
